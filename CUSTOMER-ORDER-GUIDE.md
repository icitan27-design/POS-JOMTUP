# 📱 คู่มือ: ให้ลูกค้าสแกน QR สั่งอาหารเอง + วิธี Deploy ฟรี

ฟีเจอร์นี้ให้ลูกค้าสแกน QR ที่โต๊ะ แล้วสั่งอาหารเองจากมือถือ ออเดอร์จะเข้ามาที่หน้า "บิล / คิดเงิน" ของร้านทันที พนักงานเห็นแล้วทำอาหารและเก็บเงินได้เลย

---

## 🔧 ขั้นที่ 1: เปิดสิทธิ์ให้ลูกค้าสั่งได้ (รันครั้งเดียว)

ลูกค้าไม่ได้ล็อกอิน ต้องเปิดสิทธิ์ในฐานข้อมูลก่อน:

1. เข้า Supabase → **SQL Editor** → **New query**
2. เปิดไฟล์ `supabase/customer-order-patch.sql` ในโปรเจกต์ → คัดลอกทั้งหมดไปวาง
3. กด **Run** — ถ้าขึ้น Success คือเสร็จ ✅

> แพตช์นี้เปิดให้คนที่ไม่ล็อกอิน "อ่านเมนู" และ "สร้างออเดอร์" ได้เท่านั้น ส่วนการคิดเงิน/แก้ไข/ยกเลิก ยังต้องเป็นพนักงานเท่านั้น

---

## 🌐 ขั้นที่ 2: Deploy เว็บขึ้นออนไลน์ฟรี (ด้วย Vercel)

ลูกค้าสแกนจากมือถือตัวเอง จึงต้องเอาเว็บขึ้นออนไลน์ก่อน (localhost ใช้ไม่ได้) แนะนำ **Vercel** ฟรีและง่ายสุด:

### 2.1 เตรียมโค้ดขึ้น GitHub
1. สมัคร GitHub ฟรี → https://github.com
2. สร้าง repository ใหม่ (New repository) ตั้งชื่อ เช่น `pos-restaurant` → Create
3. อัปโหลดโค้ดขึ้นไป — วิธีง่ายสุดถ้าไม่ถนัด git คือกดปุ่ม **"uploading an existing file"** ในหน้า repo แล้วลากไฟล์โปรเจกต์ทั้งหมดขึ้นไป (ยกเว้นโฟลเดอร์ `node_modules`)

> ถ้าถนัด Terminal: ในโฟลเดอร์โปรเจกต์รัน `git init`, `git add .`, `git commit -m "first"`, แล้ว push ตามคำสั่งที่ GitHub แสดง

### 2.2 เชื่อม Vercel
1. เข้า https://vercel.com → **Sign up** ด้วยบัญชี GitHub (ฟรี)
2. กด **Add New → Project**
3. เลือก repository `pos-restaurant` ที่เพิ่งสร้าง → **Import**
4. ในหน้าตั้งค่า มองหา **Environment Variables** แล้วใส่ 2 ตัว (เอาจากไฟล์ `.env` ของคุณ):
   - Key: `VITE_SUPABASE_URL` → Value: URL ของคุณ
   - Key: `VITE_SUPABASE_ANON_KEY` → Value: anon key ของคุณ
5. กด **Deploy** → รอ 1-2 นาที
6. เสร็จแล้วจะได้ลิงก์ เช่น `https://pos-restaurant-xxx.vercel.app` — นี่คือเว็บร้านคุณที่ใช้งานจริงได้!

> ทุกครั้งที่แก้โค้ดแล้ว push ขึ้น GitHub ใหม่ Vercel จะอัปเดตเว็บให้อัตโนมัติ

---

## 🖨️ ขั้นที่ 3: สร้าง QR ประจำโต๊ะแล้วพิมพ์

1. ล็อกอินเข้าเว็บที่ deploy แล้ว (ลิงก์ .vercel.app) ในฐานะ Admin
2. เมนูซ้าย ไปหน้า **QR สั่งอาหาร**
3. ช่อง **ที่อยู่เว็บของร้าน** — ใส่ลิงก์ Vercel ของคุณ (เช่น `https://pos-restaurant-xxx.vercel.app`)
   > สำคัญ: ต้องเป็นลิงก์ออนไลน์จริง ไม่ใช่ localhost
4. ใส่ **จำนวนโต๊ะ** ที่ร้านมี → กด **สร้าง QR**
5. กด **พิมพ์ทั้งหมด** → พิมพ์ออกมาตัด แปะที่โต๊ะแต่ละตัว

เท่านี้ลูกค้าก็สแกน QR ที่โต๊ะ เลือกเมนู กดสั่ง แล้วออเดอร์เด้งเข้าร้านทันที 🎉

---

## 🔄 ลูกค้าใช้งานยังไง

1. ลูกค้านั่งโต๊ะ สแกน QR ด้วยกล้องมือถือ
2. เปิดหน้าเว็บสั่งอาหาร (รู้เลขโต๊ะอัตโนมัติจาก QR)
3. เลือกเมนู กดจำนวน ใส่หมายเหตุ (เช่น ไม่เผ็ด) → กดยืนยันสั่งอาหาร
4. ขึ้นหน้า "สั่งอาหารสำเร็จ" → ทางร้านได้รับออเดอร์แล้ว
5. พนักงานเห็นบิลใหม่ในหน้า **บิล / คิดเงิน** → ทำอาหาร → คิดเงินตามปกติ

---

## ❓ ปัญหาที่พบบ่อย

- **ลูกค้าสแกนแล้วเข้าเว็บไม่ได้** → ตรวจว่า QR สร้างจาก URL ออนไลน์ (.vercel.app) ไม่ใช่ localhost
- **ลูกค้าสั่งแล้วขึ้น error** → ยังไม่ได้รัน `customer-order-patch.sql` (ขั้นที่ 1)
- **เมนูไม่ขึ้นในหน้าลูกค้า** → เมนูต้องตั้งสถานะเป็น "เปิดขาย" ในหน้าจัดการเมนู
- **อยากให้ลูกค้าจ่ายเงินออนไลน์เลย** → เวอร์ชันนี้ให้ชำระที่เคาน์เตอร์ ถ้าต้องการจ่ายผ่าน QR พร้อมเพย์ในหน้าลูกค้า แจ้งเพิ่มได้

---

## 🔒 เรื่องความปลอดภัย (อ่านไว้อุ่นใจ)

- ลูกค้า **สร้างออเดอร์ได้อย่างเดียว** — อ่านบิลคนอื่น แก้ไข หรือยกเลิกไม่ได้
- การคิดเงิน แก้เมนู ดูรายงาน ยังต้องล็อกอินเป็นพนักงาน/แอดมินเท่านั้น
- ถ้ากังวลเรื่องลูกค้าป่วน (สั่งมั่ว) แนะนำให้พนักงานกดยืนยันบิลก่อนทำอาหาร ซึ่งดูได้จากหน้าบิลอยู่แล้ว
